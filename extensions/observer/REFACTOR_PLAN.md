# S4Claw (无障碍服务 APK) 重构方案

## 当前问题分析

### 1. 前台服务问题
- ❌ 前台服务启动时机不对: 在请求权限时启动,但未进入前台模式
- ❌ 缺少前台服务状态管理
- ❌ 进程重启后前台服务状态不一致

### 2. 录屏崩溃问题
- ❌ MediaProjection 回调处理不完整
- ❌ ImageReader 初始化时机不对
- ❌ 屏幕尺寸参数获取有问题
- ❌ 缺少异常恢复机制

### 3. 界面响应问题
- ❌ 权限状态检查在主线程,每秒执行一次
- ❌ MediaProjection 状态检查涉及 Binder 调用,很慢
- ❌ UI 更新频繁导致卡顿

## 重构方案

### Phase 1: 修复前台服务 (最高优先级)
1. ✅ 移除 ForegroundService 手动启动逻辑
2. ✅ MediaProjection 授权成功后自动启动前台服务
3. ✅ 添加前台服务健康检查
4. ✅ 添加服务状态持久化

### Phase 2: 修复录屏崩溃
1. ✅ 重写 MediaProjection 回调处理
2. ✅ 添加详细的错误日志
3. ✅ 添加自动重试机制
4. ✅ 改进 ImageReader 初始化
5. ✅ 添加屏幕旋转适配

### Phase 3: 优化界面性能
1. ✅ 权限检查移到后台线程
2. ✅ 减少检查频率 (1秒 -> 2秒)
3. ✅ 使用事件驱动更新 UI (避免轮询)
4. ✅ 缓存权限状态

### Phase 4: 增强用户体验
1. ✅ 添加详细的状态说明
2. ✅ 添加问题诊断提示
3. ✅ 优化按钮状态和文案
4. ✅ 添加重试和重置选项

## 实施步骤

### Step 1: 重构 MediaProjectionHelper (核心)
- 简化 API

### Step 3: 重构 PermissionActivity
- 异步权限检查
- 事件驱动 UI

### Step 4: 测试和验证
- 测试权限授予流程
- 测试应用重启恢复
- 测试录屏稳定性
- 性能测试

- ✅ 前台服务自动启动,无需手动管理
- ✅ 录屏稳定,不再崩溃
- ✅ 界面流畅,响应迅速
